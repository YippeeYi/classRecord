📄 编日史 · 记录文件格式说明

字段说明

字段名 是否必填 说明 格式与说明
date ✅ 日期 "YYYY-MM-DD"
time ❌ 具体时间 "HH:MM"（使用半角，即英文冒号；不填时直接删除本字段）
author ✅ 记录人 姓名拼音首字母，全小写
content ✅ 记录正文 参考下文标记语法
image ✅ 原始纸质记录图片 "images/xxx.jpg"（即为将网站中图片拖动至新标签中，显示网址的末尾）
attachments ❌ 附件列表 参照样例1

正文标记语法（content）

👤 人物标记

[[zhx|qq哥]]

效果： 显示为“qq哥”，点击跳转到人物页面。

🕶 黑幕遮挡

((隐藏内容))

效果： 默认黑色遮挡，悬停显示文字。

⬆ 上标 / ⬇ 下标

H^2^O     CO_2_

效果： H²O 和 CO₂。

样例1

{
    "date": "2024-11-20",
    "author": "shr",
    "content": "[[zhx|qq哥]]将[[shr|shr]]的情书揉成一团并扔在地上，并抢夺其生物作业。[[hxw|贺宝宝]]帮助[[shr|生物课代表]]保留了情书",
    "image": "images/2024-11-20-01.jpg",
    "attachments": [
        {
            "name": "情书.jpg",
            "file": "attachments/2024-11-20-01-love.jpg"
        }
    ]
}

样例2

{
    "date": "2025-09-03",
    "time": "17:08",
    "author": "sch",
    "content": "[[cws|锁]]：[[qyc|qyc]]((鸡鸡歪歪的))",
    "image": "images/2025-09-03-02.jpg"
}

书面记录分页

书面记录页配置在 data/record/record_pages.json。图片按页码命名，放在 images/record-pages/，例如 01.png、02.png、 A1.png；每一页在 JSON 中填写 start 和 end， 值为该页展示记录的起始与结束 JSON 文件名，例如 2024-11-20-01.json。

隐藏答题题库

编日史答题页输入 lamian 后会显示 ??? 内容选项。题库填写在 data/quiz/lamian.json，图片放在 images/quiz/lamian/。 默认命名规则为 01.png、02.png、03.png， 并在 JSON 的 image 字段中对应填写完整路径。
